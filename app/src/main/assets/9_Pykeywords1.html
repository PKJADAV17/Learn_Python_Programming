• False, True, None:
    • False: Represents the Boolean value False.
    • True: Represents the Boolean value True.
    • None: Represents a null or undefined value.
  • and, or, not:
    • and: Logical AND operator.
    • or: Logical OR operator.
    • not: Logical NOT operator.
  • if, elif, else:
    • if: Used for conditional branching. Executes a block of code only if the specified condition is true.
    • elif: Stands for "else if" and is used in conjunction with if to check multiple conditions.
    • else: Part of the conditional statement and is executed if none of the preceding conditions is true.
  • for, while, break, continue:
    • for: Used for iterating over a sequence (e.g., list, tuple, string).
    • while: Creates a loop that continues until a specified condition is False.
    • break: Used to exit a loop prematurely.
    • continue: Skips the rest of the code inside a loop for the current iteration and proceeds to the next iteration.
  • in, not in:
    • in: Used to test if a value exists in a sequence (e.g., element in list).
    • not in: Used to test if a value does not exist in a sequence.
  • is, is not:
    • is: Tests for object identity (whether two variables refer to the same object).
    • is not: Tests for negated object identity.
  • def, return:
    • def: Used to define a function.
    • return: Used inside a function to specify the value to be returned.
  • class, object:
    • class: Used to define a class.
    • object: The base class for all classes in Python.
  • try, except, finally:
    • try: Used to enclose a block of code in which exceptions may occur.
    • except: Specifies the block of code to be executed if an exception occurs in the corresponding try block.
    • finally: Specifies a block of code that will be executed no matter what, whether an exception occurs or not.
  • import, from, as:
    • import: Used to import a module into a Python script.
    • from: Allows you to import specific attributes or functions from a module.
    • as: Renames a module or an imported object to avoid naming conflicts.

These are just some of the fundamental Python keywords. Understanding and using them correctly is crucial for writing effective and readable Python code.
