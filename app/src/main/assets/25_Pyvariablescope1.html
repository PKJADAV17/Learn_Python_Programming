Global Scope

Variables defined outside of any function or block have a global scope. They can be accessed from any part of the code, both inside and outside functions.

Global variables are usually declared at the top level of a script or module and are visible throughout the entire script.

Example 
global_variable = 10  

def some_function():
    print(global_variable)  

some_function()

In the example above, global_variable is defined outside any function, making it a global variable that can be accessed from within some_function.


Local Scope

Variables defined inside a function or a block have local scope. They are only accessible within that specific function or block.

Local variables are created when the function is called and destroyed when the function exits.

Example 
def some_function():
    local_variable = 20  
    print(local_variable)

some_function()

In this example, local_variable is a local variable that exists only within the scope of the some_function function.


Enclosing Scope (Closure)

In nested functions, a variable defined in the outer (enclosing) function can be accessed by the inner function. This creates an enclosing scope for the inner function.

Example 
def outer_function():
    outer_variable = 30  

    def inner_function():
        print(outer_variable)

    inner_function()

outer_function()
Output
Click on try it to see output

Here, inner_function can access the variable outer_variable from its enclosing scope, which is the outer_function.


Global Keyword

In Python, the global keyword is used to indicate that a variable is a global variable, rather than a local one, within the scope of a function. When you declare a variable as global inside a function, it means that the variable is associated with the global scope, and any modifications made to the variable inside the function will affect the global variable with the same name.

Here's a simple example.

Example 
global_variable = 10  

def modify_global_variable():
    global global_variable     
    global_variable = 20    

modify_global_variable()
print(global_variable)
Output
Click on try it to see output

In this example, global_variable is initially set to 10. The modify_global_variable function uses the global keyword to indicate that it is modifying the global variable, not creating a local one. After calling the function, the value of global_variable has been changed to 20.

It's important to note the use of the global keyword inside the function to explicitly state that the variable being modified is the one in the global scope. If you omit the global keyword, Python will treat the variable as local to the function, and any modifications will create a new local variable instead of modifying the global one.

Using the global keyword should be done with caution, as it can make the code less modular and harder to understand. It's generally recommended to minimize the use of global variables and prefer passing values through function parameters and return statements for better code organization.


nonlocal Keyword

In Python, the nonlocal keyword is used to indicate that a variable is not local to the current function's scope, but it is also not a global variable. Instead, the variable is in an enclosing (non-global) scope, typically within the scope of an outer function. The nonlocal keyword allows you to modify variables in an outer (enclosing) scope from within a nested function.

Here's an example to illustrate the use of nonlocal.

Example 
def outer_function():
outer_variable = 10  

def inner_function():
    nonlocal outer_variable  
    outer_variable = 20      

inner_function()
print(outer_variable)  

outer_function()
Output
Click on try it to see output

In this example, outer_function contains a variable called outer_variable. Inside the inner_function, the nonlocal keyword is used to indicate that outer_variable is not local to the inner_function but belongs to the enclosing scope (the scope of outer_function). The inner_function modifies the value of outer_variable, and when you print it after calling inner_function, you see the updated value.

Without the use of nonlocal, Python would treat outer_variable as a local variable within the inner_function, and modifying it would create a new local variable, not affecting the variable in the outer scope.

The nonlocal keyword is particularly useful in situations where you have nested functions, and you want to modify a variable in an enclosing scope without making it a global variable. However, like the global keyword, it should be used with caution to maintain code clarity and avoid unintended side effects.


Understanding variable scope is crucial for writing maintainable and error-free code. It helps prevent unintended variable modifications and ensures that variables are used in the appropriate context.
