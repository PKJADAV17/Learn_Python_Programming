Abstract Classes

An abstract class is a class that cannot be instantiated on its own and is meant to be subclassed by other classes.

Abstract classes in Python are created using the ABC (Abstract Base Class) module, which provides the ABC metaclass and the abstractmethod decorator.

Abstract methods declared using the @abstractmethod decorator in an abstract class must be implemented by concrete (non-abstract) subclasses.

Let's take an example of abstraction using a simple scenario involving geometric shapes. We'll create an abstract class called Shape with an abstract method area(). Then, we'll create two concrete subclasses, Circle and Rectangle, that inherit from the Shape class and implement the area() method.

Example 
from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self):
        pass

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius

    def area(self):
        return 3.14 * self.radius * self.radius

class Rectangle(Shape):
    def __init__(self, length, width):
        self.length = length
        self.width = width

    def area(self):
        return self.length * self.width

circle_instance = Circle(5)
rectangle_instance = Rectangle(4, 6)

print(f"Circle Area: {circle_instance.area()}")
print(f"Rectangle Area: {rectangle_instance.area()}")

  • In this example The Shape class is an abstract class that defines the abstract method area(). It cannot be instantiated on its own.
  • The Circle and Rectangle classes are concrete subclasses of Shape. They provide implementations for the area() method.
  • Even though Shape doesn't provide a specific implementation of area(), it enforces that any subclass must implement this method. This ensures a consistent interface for all shapes.

When you run this script, it will output the areas of a circle and a rectangle, demonstrating how abstraction allows you to create a common interface for different shapes while leaving the specific implementations to the subclasses.


Interfaces

In Python, there is no strict concept of interfaces like in some other languages (e.g., Java or C#). However, you can achieve a similar effect using abstract classes and the Abstract Base Classes (ABC) module. Abstract classes can be used to define a contract that specifies the methods that must be implemented by concrete subclasses. The ABC module provides the ABC metaclass and the abstractmethod decorator for this purpose.

Here's an example of creating an interface-like structure using abstract classes.

Example 
from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self):
        pass

    @abstractmethod
    def perimeter(self):
        pass

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius

    def area(self):
        return 3.14 * self.radius * self.radius

    def perimeter(self):
        return 2 * 3.14 * self.radius

class Square(Shape):
    def __init__(self, side_length):
        self.side_length = side_length

    def area(self):
        return self.side_length * self.side_length

    def perimeter(self):
        return 4 * self.side_length

circle_instance = Circle(5)
square_instance = Square(4)

print(f"Circle Area: {circle_instance.area()}, Perimeter: {circle_instance.perimeter()}")
print(f"Square Area: {square_instance.area()}, Perimeter: {square_instance.perimeter()}")

  • In this example The Shape class defines an interface with two abstract methods: area and perimeter.
  • The Circle and Square classes are concrete implementations of the Shape interface, providing specific implementations for both area and perimeter.

While Python doesn't enforce interfaces in the same way as languages with explicit interface support, using abstract classes in this manner helps document and ensure that classes adhere to a specific contract or set of methods. It provides a way to achieve a level of abstraction and code consistency similar to interfaces in other languages.


In summary, abstraction in Python involves creating abstract classes, abstract methods, and interfaces to simplify complex systems by focusing on essential properties and behaviors, while encapsulation helps in controlling access to the internal details of a class. These concepts contribute to building more modular, maintainable, and extensible code in object-oriented programming.
