Operator Overloading

Operator overloading in Python refers to the ability to define custom behavior for standard operators (such as +, -, *, /, ==, etc.) when applied to objects of a class. This allows you to define how objects of your class should behave when used with these operators. In Python, operator overloading is achieved by defining special methods in your class.

Here are some commonly used special methods for operator overloading.

  • __init__(self, ...): This method is called when an object is created. It initializes the object's attributes.

  • __str__(self): This method is called by the str() built-in function and should return a string representation of the object.

  • __repr__(self): This method is called by the repr() built-in function and should return an "official" string representation of the object, which can be used to recreate the object.

  • __add__(self, other): This method is called when the "+" operator is used with objects of your class. It should return the result of the addition.

  • __sub__(self, other): Similar to __add__, but for subtraction.

  • __mul__(self, other): Called for multiplication.

  • __truediv__(self, other): Called for true division (/).

  • __eq__(self, other): Called for the equality operator (==).

  • __lt__(self, other), __le__(self, other), __gt__(self, other), __ge__(self, other): Called for comparison operators (<, <=, >, >=).

Here's an example of a simple class that overloads the + operator.

Example 
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __add__(self, other):
        if isinstance(other, Point):
            return Point(self.x + other.x, self.y + other.y)
        else:
            raise TypeError("Unsupported operand type")

    def __str__(self):
        return f"Point({self.x}, {self.y})"

point1 = Point(1, 2)
point2 = Point(3, 4)
result = point1 + point2
print(result)

You can similarly overload other operators by defining the appropriate special methods in your class. This flexibility allows you to create classes that work seamlessly with Python's built-in operators.

It's important to note that operator overloading should be used judiciously and in a way that makes sense for the objects you are working with. Overloading operators can make your code more expressive and readable, but it should also adhere to common conventions to avoid confusion for other developers who may read or maintain your code.
