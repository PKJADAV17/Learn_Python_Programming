Method Overloading

While Python does not support traditional method overloading with multiple method signatures based on parameter types or number of parameters, it does provide a way to achieve a form of method overloading through default parameter values and variable-length argument lists.


Default Parameter Values

You can define a method with default parameter values. This allows the method to be called with a variable number of arguments, and if some arguments are not provided, the default values will be used.

Example 
class Calculator:
    def add(self, a, b=0, c=0):
        return a + b + c

calc = Calculator()
print(calc.add(1))         
print(calc.add(1, 2))      
print(calc.add(1, 2, 3))

In the example above, the add method can be called with one, two, or three arguments. If fewer than three arguments are provided, the missing ones are assumed to be zero.


Variable-Length Argument Lists

You can use the *args and **kwargs syntax to define functions that accept a variable number of positional and keyword arguments.

Example 
class Printer:
    def print_values(self, *args):
        for value in args:
            print(value)

printer = Printer()
printer.print_values(1, 2, 3)               
printer.print_values("a", "b", "c", "d")

In this example, the print_values method can accept any number of positional arguments. The *args syntax allows you to pass a variable number of arguments as a tuple.


Method Overriding

Method overriding is a concept in object-oriented programming where a subclass provides a specific implementation for a method that is already defined in its superclass. The overridden method in the subclass has the same name, return type, and parameters as the method in the superclass. Method overriding allows a subclass to provide a specialized implementation of a method inherited from its superclass.

Here's an example in Python.

Example 
class Animal:
    def speak(self):
        return "Generic animal sound"

class Dog(Animal):
    def speak(self):
        return "Woof!"

class Cat(Animal):
    def speak(self):
        return "Meow!"

dog = Dog()
cat = Cat()

print(dog.speak())  
print(cat.speak())

  • In this example The Animal class has a method called speak with a generic implementation.
  • The Dog and Cat classes are subclasses of Animal and both override the speak method with their own implementations.

When you create instances of Dog and Cat and call the speak method on them, the overridden methods in the subclasses are invoked, providing more specific behavior based on the type of animal.

Method overriding is crucial for achieving polymorphism in object-oriented systems. Polymorphism allows objects of different types to be treated as objects of a common type. In this case, you can treat both Dog and Cat objects as Animal objects because they share a common interface (the speak method). This promotes code reusability and flexibility, allowing you to write code that can work with objects of different subclasses in a uniform way.
