class Circle:
    def __init__(self, radius):
        self._radius = radius  
        
    @property
    def radius(self):
        """Getter method for the radius."""
        return self._radius

    @radius.setter
    def radius(self, value):
        """Setter method for the radius."""
        if value < 0:
            raise ValueError("Radius cannot be negative")
        self._radius = value

    @radius.deleter
    def radius(self):
        """Deleter method for the radius."""
        print("Deleting the radius")
        del self._radius

my_circle = Circle(5)

print(my_circle.radius)  

my_circle.radius = 10  

print(my_circle.radius)

del my_circle.radius

In this example, the property function is used as a decorator for three methods: radius (getter), radius.setter (setter), and radius.deleter (deleter). The radius property is then used to encapsulate access to the _radius attribute, allowing for validation and additional behavior when getting, setting, and deleting the radius.

Using properties helps make the code more readable, maintainable, and adheres to the principles of encapsulation. It allows you to expose a clean and consistent interface for interacting with the attributes of your class while hiding the implementation details.
