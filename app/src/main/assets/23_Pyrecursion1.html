• Base Case(s):
    • The base case is the condition under which the function stops calling itself and returns a result directly.
    • It prevents the function from calling itself indefinitely and causing a stack overflow.
  • Recursive Case(s):
    • The recursive case is where the function calls itself with modified arguments.
    • Each recursive call should move closer to the base case, ensuring that the problem becomes simpler with each iteration.

Here is the example of a factorial function to illustrate recursion.

Example 
def factorial(n):
    if n == 0 or n == 1:
        return 1
    else:
        return n * factorial(n-1)

fact=factorial(5)
print(fact)
Output
Click on try it to see output
  • In this example The base case is when n is 0 or 1, where the factorial is 1.
  • The recursive case calls factorial(n-1) and multiplies it by n.

Pros and Cons of Recursion

Following are the Pros of recursion function.

  • Clarity and Simplicity: Recursive solutions can be more elegant and easier to understand for certain problems.
  • Problem Decomposition: Recursion naturally breaks down a problem into smaller, more manageable sub-problems.

Following are the Cons of recursion function.

  • Performance Overhead: Recursive solutions may have higher memory and time complexity due to the function call stack.
  • Stack Overflow: Without proper base case handling, recursion can lead to a stack overflow if it continues indefinitely.

Recursion is a powerful tool, but it's important to use it judiciously and understand its implications on performance and memory usage.
