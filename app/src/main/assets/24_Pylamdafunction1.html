Syntax 
lambda arguments: expression

Here, lambda is the keyword, followed by a list of arguments, a colon, and an expression. The lambda function takes the arguments, performs the specified expression, and returns the result.

Let's break down the components of a lambda function.

  • lambda: This keyword is used to indicate the start of a lambda function.
  • arguments: These are the input parameters for the lambda function. You can have any number of arguments, separated by commas. The arguments are similar to the arguments you would define in a regular function.
  • : (colon): It separates the arguments from the expression.
  • expression: This is a single expression that the lambda function evaluates. The result of this expression is implicitly returned as the result of the lambda function.

Here's a simple example of a lambda function without any arguments.

Example 
# Lambda fun without arguments
greeting = lambda: "Hello, World!"

# Using the lambda fun
result = greeting()
print(result)

In this example, the lambda function doesn't take any arguments, and its expression simply returns the string "Hello, World!". The lambda function is then assigned to the variable greeting, and when called, it produces the specified greeting message.


Here's a simple example to illustrate the syntax and usage of a lambda function.

Example 
# Regular fun
def add(x, y):
    return x + y
print(add(2,3))

# Equivalent lambda fun
add_lambda = lambda x, y: x + y

# Using the lambda fun
result = add_lambda(2, 3)
print(result)

Lambda functions are often used in situations where a small, throwaway function is needed, such as when passing a function as an argument to a higher-order function like map(), filter(), or sorted().

Example 
numbers = [1, 2, 3, 4, 5]
squared = list(map(lambda x: x**2, numbers))
print(squared)  

evens = list(filter(lambda x: x % 2 == 0, numbers))
print(evens)    

words = ["apple", "banana", "cherry", "date"]
sorted_words = sorted(words, key=lambda x: len(x))
print(sorted_words)

It's important to note that while lambda functions are convenient for short operations, for more complex logic, it's often better to use a named function for clarity and reusability. Lambda functions are limited to a single expression and are generally not suitable for more extensive functionality.
