• Public, Private, and Protected Attributes:
    • Public: By default, all attributes and methods in a Python class are public, meaning they can be accessed from outside the class.
    • Private: You can use a double underscore (__) prefix to make an attribute or method private. For example, __variable or __method(). Private members are not accessible from outside the class.
    • Protected: Attributes or methods with a single underscore (_) prefix are considered protected. While they can be accessed from outside the class, it is a convention that they should be treated as if they were private. However, Python does not enforce this, and it's more of a naming convention.
  • Accessing Encapsulated Members:
    • Public members can be accessed directly: obj.public_member.
    • Private members can be accessed within the class itself, but attempting to access them from outside the class will result in an AttributeError.
    • Protected members follow a similar pattern as private members but are often accessible from outside the class, although this is discouraged.
  • Getter and Setter Methods:
    • To provide controlled access to private members, classes often define getter and setter methods.
    • A getter method retrieves the value of a private member, while a setter method allows you to modify its value.

Here's a simple example to illustrate encapsulation in Python.

Example 
class Car:
    def __init__(self):
        self.__speed = 0  

    def accelerate(self):
        self.__speed += 5

    def get_speed(self):  
        return self.__speed

    def set_speed(self, speed):
        if speed >= 0:
            self.__speed = speed

my_car = Car()
my_car.accelerate()
print(my_car.get_speed())  
my_car.set_speed(20)       
print(my_car.get_speed())

In this example, __speed is a private attribute, and its value is controlled through getter and setter methods. This encapsulation helps in maintaining control over the internal state of the Car class.
